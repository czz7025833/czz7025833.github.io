如何使用Hexo和Github Pages免费搭建博客

Hello，这是我的第一篇博客，写的就是关于如何搭建起这个博客的过程，如何使用Hexo和Github Pages来免费的完成博客的搭建。以后我也会尽量多更新一些博客，记录我做为一个技术人员的日常的点点滴滴。

什么是Hexo

hexo的官网地址 https://hexo.io/
hexo简单的说就是一个静态博客网页的生成器，使用Node.js，搭建起来便捷，快速，也简单，同时也方便一键部署到别的地方，就比如github pages。

什么是Github Pages

github pages的官网地址 https://pages.github.com/
github pages简单来说就是github帮助你显示你准备好的静态网页，那前提也就是你要把你的网页项目放在github上面。
doc页面 https://help.github.com/en/github/working-with-github-pages

如何使用Hexo生成静态的博客

  1. 安装Node.js

    hexo是基于Node.js的，所以第一步需要配置好Node.js的环境，我的机器是Mac，可能以前已经装过Node.js了，可以直接查看版本

    如果没有安装过，mac可以使用

    如果是其他的系统，直接参考Node.js的官网，下载对应的版本， https://nodejs.org/zh-cn/

  2. 安装hexo cli

    hexo cli就是一个方便你使用hexo的工具，能够帮你做一系列初始化的工作。直接使用npm来安装就行

    可以看到，失败了，gyp出现了错误，介于我对于Node.js方面也是小白一枚，所以只有果断百度， 说是更新一下npm就行

    然后重新安装hexo-cli，这次安装成功了，但是后面还是报了别的错

    那这个报错信息也就比较明显了，需要的库需要node版本大于8，现在本地的node版本只有6.10.3，所以先要升级node的版本，继续百度，提供一篇找到的博客，说如何控制node版本的文章， https://www.cnblogs.com/chaoyangya/p/10484513.html

    先安装 node 的版本控制器, 安装stable版本的Node.js

    安装完成之后，直接n，进入n版本控制器，选择对应的node版本，然后安装就行了，最后重新更新一下hexo-cli

    大功告成，也是好几番波折，如果大家一开始重新安装最新版本的Node.js的话，就不会有后续的问题

  3. 初始化和配置Hexo

    初始化hexo的项目，从log中基本可以看出，初始化其实就是从git上面clone下项目，然后安装依赖

    从浏览器打开默认的地址 http://localhost:4000

    就可以看到它默认主题的博客网页的，成功！！！！

如何把Hexo生成的静态网页部署到github pages上面

  1. 安装hexo中的hexo-deployer-git插件

    在_config.yml文件中, 配置插件的参数，主要就是git相关的参数

    首先需要github的账号，这个应该没啥问题，同时如果要使用github page的话，就需要创建一个名字为{username}.github.io的repository，这个repository会默认为github page的主仓库，注意只有github pro和public的repository才能使用github page，私有的仓库是不行的，我一开始设为私有，它需要你改为公有，或者交钱升级到pro才行。

    但是没有成功，输入网址czz7025833.github.io， 显示并没有对应的github的page，可能需要等github 配置一下， github的帮助上有这样写道：

    Note: It can take up to 20 minutes for changes to your site to publish after you push the changes to GitHub. If your don’t see your changes reflected in your browser after an hour, see “About Jekyll build errors for GitHub Pages sites.”

    等了一段时间之后，网站就成功上线了

    https://czz7025833.github.io/
